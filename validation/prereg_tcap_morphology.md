# Предрегистрация №1 (Etruscan): перенос MDL-морфологии на Капуанскую табличку

Дата заморозки правил: 2026-07-11. Ритуал по образцу LA-ветки
(CROSS_PROJECT, записи 2026-07-10/11): правила и скрипт коммитятся ДО
первого прогона; прогон ОДИН; результат публикуется как вышло.

## Гипотеза

H1 (первичная): суффиксальная грамматика, выученная MDL-алгоритмом
(tools/etr_mdl_morph.py, §9.4) на каноническом виде корпуса v0.8 **без
Капуанской таблички** (artifact_id != 'ART:TCap'), продуктивна на её
словаре: доля НОВЫХ типов TCap (len>=3, без '-', не встречающихся в
train-типах), разложимых как train-основа + непустой суффикс из S,
превышает нуль случайных суффикс-наборов.

H2 (вторичная, направление регистрируется): биты/токен MDL-модели на
токен-потоке TCap МЕНЬШЕ, чем у посимвольной 3-граммной LM (B1, те же
train-данные). Публикуется при любом исходе.

## Замороженная процедура

- Корпус: data/etr_corpus.pkl, freeze v0.8 (sha 01f2cec9…), канонический
  вид (etr/text/без forgery?/variant_of None).
- Train: все записи вида с artifact_id != 'ART:TCap'. Test: записи
  ART:TCap (75 строк CIEW 7002 + возможные ETP/CIEP-дубли той же таблички
  исключаются из train автоматически через artifact_id).
- MDL-машинерия и параметры БУКВАЛЬНО из tools/etr_mdl_morph.py:
  MIN_TYPES=15, MAX_SUF=60, MAX_SUF_LEN=4, MIN_STEM=3, 3 раунда
  переназначения, SEED=42.
- Нуль H1: R=200 случайных суффикс-наборов той же мощности и того же
  мультимножества длин из пула кандидатов (конструкция §9.4 (b)).
- Критерий успеха H1: p = (1+#{null>=obs})/(R+1) <= 0.05.
- Критерий H2: bits/token(model) < bits/token(B1) — бинарно, без p.
- Скрипт: tools/etr_prereg_tcap.py; прогон один; лог коммитится как есть.

## Зарегистрированные оговорки

- TCap несёт OCR-шум CIEW-слоя (sample50 #27; §0.25) — шум размывает
  разложимость К НУЛЮ, т.е. тест консервативен к H1.
- Жанровый сдвиг (ритуальный календарь против преобладающего
  погребально-посвятительного train) — часть проверяемого утверждения:
  морфология должна быть жанрово-инвариантной, лексика — нет.
- Никакие параметры/пороги после прогона не меняются; повторных прогонов
  с иными настройками не будет (негатив публикуется).
