# Этрусский язык: статистическое расширение понимания — отчёт

*Сестринский проект DecipherLinearA (DOI 10.5281/zenodo.21262275);
методическое ядро — оттуда: замороженный валидированный корпус, явные
нуль-модели (перестановки, seed=42), семейный контроль множественных
сравнений, честные негативы. Отличие от LA: существует валидационный
набор (≈33% записей канонического вида с переводами), поэтому все методы «значений» обязаны
отчитываться метрикой на отложенной выборке.*

*Все прогоны: `PYTHONHASHSEED=0`, `PYTHONIOENCODING=utf-8`, python 3.12
(`uv venv`); замороженный аналитический pipeline даёт пустой `git diff`
после полного перепрогона. Сетевые refresh-скрипты отдельно фиксируют время
получения и точные upstream revisions.*

> **Аудит 2026-07-10 имеет приоритет над историческими разделами ниже.**
> Westfall–Young `p̃` в operator/second-position/LL коде отозваны;
> положительная атрибуция Лемноса аннулирована из-за target leakage;
> индивидуально устойчивы 7, не 10 региональных основ. Канонические числа,
> нули и ограничения собраны в §8 и `INPUT_FROM_SOL.md`.

## §0. Заморозка корпуса (etr_corpus.pkl, нормализация v0.1)

### 0.1. Источники

| файл | строк данных | что это |
|---|---|---|
| `etruscan_larth.csv` | 7139 | компиляция Larth-Etruscan-NLP (Vico & Spanakis 2023, ACL ALP): ID, город, датировка (годы до н.э.), текст, англ. перевод |
| `ETP_fix.csv` | 561 | ETP-данные из того же репозитория (контрольный источник) |
| `CIEP_pymupdf.csv` | 5833 | pymupdf-извлечение конкорданса CIEP: `key`, `T` (дипломатическое чтение), `C` (исправленное), `A` (глоссы), `CIEP` (номер записи) |

Числа строк здесь — без заголовка (в CLAUDE.md они на 1 больше). README
репозитория Larth конвенций транслитерации НЕ документирует — они
восстановлены профилированием и внешней сверкой (см. 0.4, 0.7).

**Установленное устройство компиляции** (проверяется скриптом заморозки
при каждом прогоне, нарушение = assert):

1. `etruscan_larth.csv` = ETP-часть (561 строка, `key` пуст) + CIEP-часть
   (6578 строк, `key` непуст).
2. ETP-часть **дословно содержит все 561 строку `ETP_fix.csv`** (по
   ID+тексту; несовпадений 0).
3. Каждая строка CIEP-части — **проекция колонки T или C** записи
   `CIEP_pymupdf.csv` с тем же `(CIEP-номер, key)`: текст=T — 4193,
   текст=C — 2385, не сопоставлено — 0. Т.е. larth-компиляция не содержит
   для CIEP-части ничего сверх исходного конкорданса, а пары строк T+C
   одной записи — это **два чтения одной записи, а не две надписи**.

### 0.2. Единица корпуса и дедупликация

- **ETP-часть:** запись = уникальная пара (ID, текст); 561 строка → **528
  записей** (33 строки-дубля слиты; влито 8 вариантов перевода — напр.,
  `LL 8` имел два перевода одной строки). Один ID может иметь несколько
  записей-текстов (`rid` вида `ETP:LL 6#2`): это варианты чтения или
  фрагменты одного памятника (см. негатив N4 в 0.9).
- **CIEP-часть:** запись = строка исходного `CIEP_pymupdf.csv` (**5833
  записи**); предпочтённое чтение C, иначе T; оба raw-чтения сохранены
  (`raw_T`, `raw_C`). Двойные чтения T&C — 1159 записей — слиты **по
  построению**, что устраняет двойной счёт, который дают сырые строки
  larth (пилот §BM считал по строкам — его числа завышали корпус).
- Итого **6361 запись**; `rid` уникальны (проверка: 0 дублей).

### 0.4. Нормализация v0.1 (полные правила — докстринг `etr_freeze.py`)

Разметка источника (лейденские конвенции) разбирается, а не стирается:

| разметка | смысл | действие |
|---|---|---|
| `[абв]` | реставрация в лакуне | текст остаётся, флаг `restored` (292 токена) |
| `[---]`, `[…]`, `[8-10 letters]` | лакуна | токен-гэп `kind=G` (38) |
| `[vacat]` | пустое место на камне | убирается (не лакуна) |
| `<абв>` | эмендация (пропуск писца) | остаётся, `emended` (71) |
| `(абв)` | раскрытие аббревиатуры | остаётся, `expanded` (1913) |
| `{абв}` | лишние буквы писца | вычёркивается, `scribal_extra` (127); НО если после вычёркивания букв не остаётся (CIEP-записи целиком в скобках — семантика конкорданса неясна), текст сохраняется с флагом `braced` (11) |
| точки при буквах, подстрочные точки U+0323 | неуверенное чтение | точки убраны, `uncertain` (47) |
| прогоны `--` | повреждение | остаются в форме, `damaged` (592) |
| `/` внутри слова | перенос строки | склейка, `line_split` (6) |
| `\|`, отдельные `/` | граница строки надписи | счётчик `n_lines` |
| `•` `:` `;` отдельные `.` | разделители слов | границы токенов |
| токен из IVXLC с заглавной | числительное | `kind=N`, 26 токенов (напр. `avil CVI` — ветеран из Капуи, 106 лет: разобрано верно) |

Прочее: html-сущности (`&gt;`) декодируются; NFC; `’/ʼ→'`; `Σ→σ`;
одиночный дефис в CIEP-части — разделитель слов конкорданса (33 разреза;
в ETP-части дефис-разделитель не используется — проверено). Графемы
**не сливаются**: σ, σ', ś, š, ς различаются источником и сохранены как
есть; **ê — особый знак Кортонской таблички** («перевёрнутый эпсилон» ɜ
изданий) — встречается только в записях `ETP 74` (проверка 0.7.4).
Для кросс-сверок частей корпуса есть ascii-проекция формы (θ→th, χ→ch,
φ→ph, σ/ς/ś/š→s, ê→e) — с потерями, в морфологии не используется.
Проверка непотери букв: буквенный состав raw == буквенный состав токенов
в **6361/6361 записей (100%)**. Вычищено: 1 токен без букв, символы
`1`, `?` (по одному разу).

Профилирование и ручной просмотр выборки выявили в CIEP-части
неэтрусский материал; языковые метки назначены задокументированными
правилами (обоснования — в `LANG_BY_CIE/LANG_BY_ETP` скрипта):

| lang | записей | что это |
|---|---|---|
| `etr` | 6240 | основной вид |
| `umb` | 95 | Игувинские таблицы, умбрский (ikuvina, esunu, fertu) |
| `lat` | 13 | латинские: ETP `Pe 1.211`, `Ar 1.3`, `Um 1.7` (билингва гаруспика) + CIEP 17299 (`de praidad`), 17181 (`romanom`, монеты), 17286 (`duonoro(m)`, эпитафии Сципионов) |
| `cel` | 11 | цизальпинско-кельтский, камень Брионы (kuitos lekatos) |

### 0.7. Внешняя валидация

**ETP-онлайн (etp.classics.umass.edu) недоступен** (ECONNREFUSED),
ручная сверка 50 записей с БД невозможна — честная замена:

1. **Cippus Perusinus** (транскрипция Roncalli): из 19 текстовых записей
   `CIEP:4538:*` — 17 находятся подстрокой опубликованного текста
   (сравнение по ascii-сквошу токенов, издательские конвенции и
   словоделение нейтрализованы), 2 — задокументированные варианты
   изданий (`rezus` — повреждённая строка 1; `finθa`/`hinθa` — мена f/h).
2. **Tabula Cortonensis** (секция I): полный текст таблички лежит в
   `ETP:ETP 74#6`; 32/34 слов издания найдены; 2 варианта изданий
   (`clθii`/`clθil`; реставрация `tɜnθa[ś]`).
3. **Liber Linteus col. III** (van der Meer 2007) — слабая проверка (в
   корпусе другие строки LL): пересечение лексики — формульное ядро
   `cletram, fler, śrenχve, trin`.
4. **Графема ê** встречается только в `ETP 74` = Кортонская табличка —
   подтверждает отождествление ê = ɜ.

**Вердикт автоматической сверки: 0 необъяснённых расхождений** в перечисленных
выше опубликованных фрагментах. `validation/sample50.md` (50 записей,
seed=42) сгенерирована, но внешние поля проверки пока заполнены для **0/50**;
она не считается пройденной. Полная ручная сверка по изданиям остаётся
P0-задачей для владельца проекта.

### 0.9. Негативы и ограничения

- **N1.** Скрижали Пирги в корпусе отсутствуют; знаменитые большие
  тексты представлены неполно (LL — 12 строк-цитат; Tabula Capuana —
  только конкорданс-слова 8682). → supplement-кандидаты.
- **N2.** CIEP-часть — словный конкорданс с ненадёжной сегментацией:
  654 записи — слитные строки ≥12 букв (`seg=long-unseg`, напр.
  `tesnsteisrasneschimthsp`); города и датировки в CIEP-части
  отсутствуют полностью (метаданные есть только у ETP-части: 456
  городов, 358 датировок). Репликация «по городам и векам» (план §3)
  реально выполнима только на ETP-подмножестве.
- **N3.** Остаточная языковая контаминация CIEP-части не исключена
  (фалискский/латынь без явных маркеров); митигируется репликацией на
  ETP-части (0.5).
- **N4.** Варианты чтения одного памятника живут отдельными записями
  (`ETP:LL 6`, `#2`, `#3`: luśao/luśaσ/luśaσ' — одна строка LL в трёх
  чтениях) — лёгкая инфляция типов; сливать без изданий нельзя, отмечено
  для этапа 5.
- **N5.** Переводы CIEP-части — склеенные глоссы (`of-the-goddess-vipsi`,
  `mr-`/`mrs-` = праеномен-плейсхолдеры), для supervised-семантики (§4)
  потребуют отдельной очистки; качество ниже ETP-переводов.
- **N6.** Конвенции транслитерации Larth не опубликованы — наши правила
  v0.1 восстановлены из данных и внешней сверки; при появлении
  документации возможна ревизия (версия нормализации хранится в meta
  корпуса).

### 0.13. Находка Fowler & Wolfe 1965: путь к Liber Linteus и полному CIE

На archive.org найден OCR-слой тома I «Materials for the Study of the
Etruscan Language» (Fowler & Wolfe 1965; «CIEW») — машинный компендиум:
**CIE 1–5606 и 8001–8464, TLE (вкл. Капуанскую черепицу), ПОЛНЫЙ Liber
Linteus (запись 9001, версия Runes), Лемносская стела (9002), печень
Пьяченцы (9003), Пирги (9021)**. Копия и черновая таблица декодирования —
`data/external/fowler_wolfe/` (README; правовой статус: копирайт 1965,
по утверждению загрузившего не продлён — проверить перед публикацией).

Пилотная оценка извлекаемости: формат строк «номер-записи номер-строки
текст» парсится; кодировка восстанавливается (θ→6/9, χ→X, f→8/F);
контрольные совпадения — col. III LL дословно = van der Meer 2007,
`8LU 6UPIT` = CIEP «fluthupit». **Вердикт: полный парсер CIEW — реальный
следующий этап корпуса (v0.4)**; уникальная возможность валидации — CIE-
номера CIEW и CIEP-части совпадают, пословная сверка даст количество
OCR-ошибок. Это снимает главное узкое место проекта (машиночитаемый LL).

## §2. Калибровка морфем

**Артефакты:** `tools/etr_morphemes.py` → `results/morphemes_v1.csv`,
лог `logs/etr_morphemes.log`. Вид v0.3 (5858 записей); ярус A — весь
корпус в ascii-проекции (5074 типа), ярус B — богатые формы ETP-части
(1147 типов, сибилянты различены).

### 2.1. Методика

Скрининг всех концовок длины 1–3 с ≥15 типами. Статистика «поддержка»:
типы, чья основа (слово минус суффикс) сама в словаре. Нуль — позиционная
перестановка хвостов при фиксированных основах, что даёт **точный
гипергеометрический нуль** (без МК); семейная поправка Бонферрони
(m=115). Парный тест парадигм: основы с двумя суффиксами, двусторонний
гипергеометрический нуль (m=91×2). Семантические тесты — перестановка
глосс, R=10000, seed=42.

### 2.2. Скрининг: значима только энклитика -c

Единственная концовка, пережившая Бонферрони по «поддержке голой
основой»: **-c** (97 типов, поддержка 35 при E=13, обогащение 2.7×,
p_adj=1.2e-06) — энклитика «и», основы которой — полные слова, что тест
и ловит. Семантически подтверждена глоссами: -c ↔ «and», p=0.0001
(точность 25% при базе 1%). Для падежных суффиксов тест «голой основы»
слаб по построению (номинатив имени часто не засвидетельствован).

### 2.3. Главный результат: комплементарность падежных маркеров

Парный тест: **0 значимо обогащённых пар** (см. N11), но **10 значимо
комплементарных** (депрессия пересечения, Бонферрони ×182):

| пара | основ A | основ B | ∩ набл. | ∩ ожид. | p |
|---|---|---|---|---|---|
| -s / -al | 1105 | 598 | 16 | 177 | 2.9e-75 |
| -s / -l | 1105 | 677 | 40 | 200 | 1.2e-62 |
| -l / -al | 677 | 598 | 3 | 108 | 4.4e-52 |
| -s / -ial | 1105 | 453 | 16 | 134 | 4.5e-51 |
| -al / -sa; -s / -na; -l / -sa; -ial / -sa; -s / -isa; -l / -ial | | | | | ≤6.6e-16 |

Основные именные концовки разбивают пространство основ на почти
непересекающиеся классы. **Поправка аудита 2026-07-10:** исходный all-match
двойным образом относил `X-ial` к `-ial/-al/-l` и `X-isa` к
`-isa/-sa/-s`. При longest-match и полном контроле строго сохраняются 5 из
10 перечисленных пар; ключевые `-s/-al` (`p_Bonf=1.67e-5`, source forms) и `-s/-l`
устойчивы, вложенные `-l/-al` и `-l/-ial` исчезают. Массовая депрессия
27/86 невложенных пар указывает на misfit простого hypergeom-null/общие
лексические классы. Итог: сильное свидетельство классов `-s` против
`-al/-l`, но не «доказательство падежа» для каждой из десяти пар без
held-out морфологической модели.

### 2.4. Сибилянтный ярус и негативы

Ярус B: -s (235 типов) и -σ/-ś/-š (69) поддержки голой основой не
показали (p=0.98/0.71) — та же причина, что в 2.2. **N11:** парадигмы
одного типа (основа с двумя падежами) на уровне корпуса типов НЕ
обнаружимы — стена гапаксов (84%); индивидуальные парадигмы живут внутри
больших текстов (velθinas/velθina в CP), не между надписями. **N13:**
вековая репликация слаба — датированы только 308 записей ETP-части.

### 2.5. Семантика морфем по глоссам (1533 одно-словных переведённых)

| морфема | тест | точность | база | p |
|---|---|---|---|---|
| -s | генитив глоссы («of-…») | 15% | 7% | **0.0001** |
| -(i)sa | посессив/супружество | **84%** | 15% | **0.0001** |
| -ce | глагольная глосса (англ. прош.) | 20% | 2% | **0.0003** |
| -c | «and» | 25% | 1% | **0.0001** |
| -al | генитив глоссы | 6% | 7% | 0.69 (N12) |

**N12:** -al провалил глоссовый тест; возможное объяснение — конвенция
Хилла, где матронимы глоссируются «mrs-X» без «of». Гендерный тест ниже
не является независимым подтверждением: он использует те же глоссы.

**Пол имени** (глоссы mr-/mrs-: 540 муж., 253 жен.) даёт крупные
описательные контрасты: женские концовки **-i (39% против 7% у мужских),
-l (34%/10%), -al (33%/9%), -ei (11%/1%)**; мужские **-s (26%/2%), -as
(9%/0%), -es, -u, -e**. Однако код выбрал top-10 из 18 концовок по
наблюдаемому |Δ| и поправлял лишь выбранные десять; кроме того, старый W–Y
невалиден. Например, для -a raw `p=.0049`, но Bonferroni ×18 даёт `.0882`.
Следовательно, общая гендерная организация описательно намечается, но требует
заранее заданного теста по всем 18 окончаниям; ни одна выбранная post hoc
концовка не считается установленной прежним тестом.

### 2.6. Контроль: комплементарность — грамматика, не фонотактика

Старый стратифицированный депрессионный тест
(`tools/etr_morph_controls.py`, MC R=10000) считал вложенные окончания
одновременно и повторно проверял 10 пар, выбранных из 91 на тех же данных.
После longest-match, полного Bonferroni ×91 и фонотактического контроля
строгими остаются **5/10** первоначальных пар. Ключевой результат -s/-al:
5 общих основ при `E=21.42`, raw `p=1.83×10⁻⁷`, Bonferroni
`p=1.67×10⁻⁵`; -s/-l также устойчив. Sensitivity без `ocr?`-records и
редакторски сомнительных токенов сохраняет те же 5/10; для -s/-al obs=4,
`E=15.77`, `p_Bonf=.00178`. Но подавлены 27/86 невложенных пар,
что показывает misfit простого hypergeometric null/лексические классы.
Комплементарность реальна, а её интерпретация как падежа или рода требует
независимой парадигматической и контекстной проверки.

### 2.7. Вывод §2

На текущем уровне статистически поддержаны ассоциации -c с «and», -(i)sa с
посессивно-супружескими глоссами, -ce с глагольными и -s с генитивными
глоссами, а также несколько устойчивых комплементарных пар окончаний.
Последние сами по себе ещё не доказывают падежные классы; гендерные контрасты
требуют повторного заранее заданного теста по всем 18 концовкам. Карта
морфем v1 в `results/morphemes_v1.csv` остаётся реестром гипотез, а не
словарём доказанных значений.

## §3. Семантика с учителем v1: классы слов

**Артефакты:** `tools/etr_semantics.py` →
`results/semantic_hypotheses_v1.csv`, лог `logs/etr_semantics.log`.

> **AUDIT OVERRIDE 2026-07-10.** Разделы v1–v7 ниже описывают development
> experiments: один fixed split многократно использовался при выборе моделей
> и больше не является независимым test. В v1/v2 также найден calibration bug
> (`empirical_accuracy == 0` заменялось raw confidence); код исправлен, но
> исторические scores от этого не становятся external calibration. Поэтому `prob_cal`
> нельзя трактовать как вероятность значения, а списки — только как очереди
> кандидатов. Независимая cross-source проверка и новый context-only пилот
> приведены в §8.2.

### 3.1. Постановка и данные

Предсказываем СЕМАНТИЧЕСКИЙ КЛАСС типа слова (не перевод). Разметка — из
глоссов одно-словных переведённых записей по фиксированным априорным
правилам (маркер лица mrs-/ms-/mr- по самой ранней позиции в глоссе —
покрывает генитивы «of-mr-…»; затем THEO/KIN/VERB/OTHER): **1305
размеченных типов** (NAME-M 562, OTHER 416, NAME-F 242, THEO 65, VERB 20;
KIN(5)→OTHER по заранее объявленному правилу n<15). Признаки без утечки
переводов: конечные n-граммы 1–3, начальные 1–2, внутренние биграммы,
длина, частота, регион, соседи-операторы §1, относительная позиция.
Модель: биномиальный наивный Байес (numpy). Оценка: стратифицированный
train/test 80/20 по типам, seed=42.

### 3.2. Качество (отложенные 260 типов)

**Точность 60.0%** против: мажоритарного 43.1%, правила гендерных
суффиксов §2 44.2%, перестановочного нуля 40.1% (max 50.8% на R=1000)
— **p=0.001**. По классам: NAME-M F1=0.64, NAME-F F1=0.63 (полнота 79%),
OTHER F1=0.58; **THEO и VERB — F1=0 (N14)**: теонимы морфологически
неотличимы от имён (лингвистически ожидаемо — нужен контекст записи, а
не форма слова), глаголов слишком мало (20 типов).

### 3.3. Историческая калибровка (не подтверждена аудитом)

Сырые вероятности NB самоуверенны; историческое отображение «уверенность →
эмпирическая точность» давало в верхнем бине 66.3% против наблюдаемых 64.3%.

### 6.1. Классификатор языка: лемнийский не атрибутируется — и это ответ

## §7. Концепт-лексикон эпохи и фонетический зонд заимствований

**Идея владельца проекта; артефакты:**
`data/concepts/concept_lexicon.csv` — ПЕРЕНОСИМАЯ структура: 230
понятий эпохи бронзы/железа × 7 расшифрованных языков (греч., лат.,

мин. длина 3); понятия с конвергентными формами между языками (нуль —

### 8.1. Единица независимости и качество наблюдений

В актуальном виде 70.6% records однословны; 80.2% переведённых records также
однословны. CIEP — конкорданс, а не набор независимых документов: 5363

следующий формат должен представлять варианты как lattice и прогонять Monte

После кластерных нулей просопографический MI остаётся (`p=.0005`), но

record-null; условная перестановка внутри geography ослабляет её до
`p=.0225`, Monte Carlo по интервалам дат даёт MI median .568, 95%
[.540, .608].

### 8.2. Новый context-only пилот

PPMI/centroid classifier полностью исключает CIEP и признаки формы; gold —
только ETP_POS, tied gold исключён, case-stripped families не пересекают folds.

разницы), balanced accuracy **.391** против family-block перестановочного нуля
.198 (**p=.0005**, R=2000, сохранены class marginals и внутрисемейная

### 8.4. Raetic → Etruscan case transfer

Это первичная **exploratory** validation: протокол не был preregistered, а
четыре inferential tests и их Bonferroni-family зафиксированы в выходном JSON
для воспроизводимости, не для ретроспективного объявления confirmatory study.

Categorical NB с **uniform class prior** обучен только на 90 полнословных Raetic entries TIR;
признаки — финальные 1–4-граммы. Без refit он проверен на 506 attested,
non-suffix ETP_POS-типах с хотя бы одним token-clean вхождением вне `ocr?` record актуального
корпуса; 6 типов, встречающихся только в помеченных чтениях, исключены. Если ни одна
обучающая 1–4-грамма не встречена, модель теперь явно воздерживается:

- покрытие **435/506 = 86.0%**; на покрытых типах accuracy **.676**,
  balanced accuracy **.756**, macro-F1 .613;
- точное пересечение нормализованных Raetic train и Etruscan test surfaces:
  **0**;
- conditional target-family bootstrap 95% CI balanced accuracy
  **[.714, .795]**; TIR train в нём фиксирован;
- lemma-block permutation сохраняет class marginals и внутри-lemma vectors:
  **p_raw=.00070**, после Bonferroni по четырём тестам **p=.00280**;
- marginal-preserving target-family block permutation (412 семейств):
  **p_raw=.00010**, **p_Bonferroni×4=.00040**;
- GEN precision/recall .878/.721; NOM .661/.589; PERT .274/.958.

Descriptive sensitivity с empirical TIR class prior не ослабляет, а усиливает
минимальную нормализацию: accuracy **.708**, balanced accuracy **.780**
(strict: .663/.746). Это не дополнительный confirmatory test.

Строгая графемная абляция также даёт сигнал: покрытие **356/480 = 74.2%**,
balanced accuracy **.719**. Sensitivity-модель только на 66 TIR entries с
`checklevel=0` покрывает 420/506 и даёт balanced **.723**; это описательная
проверка устойчивости, не пятый тест в заявленном семействе четырёх.

После основного анализа выбрана **post-hoc** абляция, использующая ровно один
самый длинный известный суффикс длины 2–4 и не суммирующая вложенные n-граммы.
Она покрывает 258/506 (51.0%): accuracy .888, balanced .896. На subset,
совпадающем с candidate eligibility по `freq_clean>=2`, `n=102`, accuracy
.931, balanced .939; matched precision GEN .938 (61/65, Wilson 95% CI
[.852,.976]), NOM 1.000 (31/31, [.890,1.000]), PERT .500 (3/6,
[.188,.812]). Именно эта post-hoc калибровка приложена к 265 строкам
candidate CSV (GEN 127, NOM 118, PERT 20).
Из них 23 уже присутствуют в иных, исключённых из clean single-case gold,
строках ETP_POS; это явно отмечено `already_present_in_etp_pos`.
Record-level `ocr?` исключён из clean frequency, а aliases Liber Linteus и
Tabula Capuana объединены между ETP/CIEP/CIEW. Дополнительные колонки
показывают реальную опору: 133/265 кандидатов имеют TIR-суффикс с train
support=1, 59/265 засвидетельствованы только в одном artifact-cluster; оба
минимальных фильтра support≥2 проходят **91/265**.

Результат показывает измеримое согласие прежде всего номинальной и
ономастической суффиксации, но не является независимым доказательством
родства или всего падежного строя: 86/90 обучающих форм — proper nouns, а TIR
сам использует этрусские сравнения. Candidate CSV — exploratory очередь без
индивидуальных p-values/FDR; `score_margin_nonprobabilistic` есть разность
сглаженных log-likelihood, не вероятность. Поэтому строки нельзя читать как
установленные падежи или переводы.

### 8.5. Итог аудита

Сильная цель — полный перевод произвольного текста — ограничена не моделью,
а информацией: 81% типов гапаксы, большая часть материала однословна,
переводы и POS частично происходят из одной традиции, длинного независимого
билингвального gold нет. На текущем корпусе реалистична слабая цель:
artifact-robust морфология, структура жанров/периодов, проверяемые классы слов
и внешне ранжированные кандидаты. Наиболее перспективны три отличных от
старого pipeline подхода: artifact graph + reading lattice; held-out MDL/FST
морфология; context-only anchor graph с latent period slots. Дополнительные
проверяемые benchmarks — object/formula-blocked restoration и USEP
object/iconography graph. Полные критерии и порядок работ записаны в
`INPUT_FROM_SOL.md`.

### 9.5. Anchor graph (план Sol №3, часть 1): распространение по графу НЕ бьёт центроид — негатив

propagation (α=0.85, 30 итераций) поверх 178 ETP-якорей (NAME-M/NAME-F/
THEO/VERB/FUNC), те же family-blocked 5 фолдов и нуль Sol (перестановка
векторов меток между семьями равного размера, R=1000).

(p против семейного нуля 19.7% всё ещё 0.001), macro-F1 0.276 против
0.356, VERB F1 0.459 против 0.500** — NAME-M (88/178 якорей) затопляет
граф, NAME-F гибнет (F1 .407→.000). Диагноз содержателен: kNN-граф на
окне ±2 кодирует СЛОТОВОЕ соседство ономастической формулы, где мужские

ровно то различие (род), которое центроид ещё видел; контроль «центроид
на граф-признаках» держит 34.4%, т.е. виновата динамика распространения.
Вывод: на 178 якорях многошаговая транзитивность добавляет шум;
центроидный перенос Sol остаётся лучшим context-only методом. Негатив
публикуем; гипотезный слой (`results/anchor_graph_hypotheses_v1.csv`,
топ-300 по margin) осмыслен только как генератор кандидатов NAME-M/VERB
(CV-precision .56/.58) с центроидной перепроверкой. Вторая часть плана
№3 (semi-Markov по периодам LL) разблокирована §9.1 — следующий этап.
