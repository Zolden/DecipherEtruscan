# DecipherEtruscan — инструкции проекта

## Миссия
Расширить ПОНИМАНИЕ этрусского языка статистическими методами: полная
формальная грамматика, вероятностные значения для неизвестных слов
(дистрибутивная семантика с учителем), структурное чтение больших текстов
(Liber Linteus). Мы НЕ обещаем полную дешифровку; каждое утверждение —
«кандидат / класс / согласуется с», с перестановочными нуль-моделями.

## Происхождение и сестринский проект
Проект отпочкован от **DecipherLinearA** (C:\OtherProjects\DecipherLinearA;

**Кросс-проектный протокол:** если здесь обнаружится что-то полезное для
линейного A (методическая находка, данные, параллель эгейско-тирренской
гипотезы — например, финальные гласные, биморфы, слот-структуры), запиши
это в CROSS_PROJECT.md этого репозитория И скажи пользователю, что находку
стоит передать в тред DecipherLinearA. Аналогичный файл есть на той стороне.

## Дисциплина метода (унаследована, обязательна)
1. Корпус замораживается и валидируется; corpus-файл НЕ редактируется
   (добавления — через supplement-механизм).
2. Каждая закономерность — против явной нуль-модели (перестановка меток,
   позиционная перестановка, ресемплинг); seed=42, R=10000 (или указать);
   p — разведочные; для семейств кандидатов — семейный p_max.
3. PYTHONHASHSEED=0 во всех прогонах; логи коммитятся; критерий
   воспроизводимости — пустой git diff после полного прогона.
4. Негативы публикуются наравне с позитивами; свои ошибки — исправлять
   явно (retract с пометкой).
5. Никаких «переводов» сверх проверяемого; НО: в отличие от LA, здесь есть
   валидационный набор (≈42% надписей с переводами) — методы «значений»
   обязаны отчитываться метрикой на отложенной выборке.

## Пилотные факты (уже установлено, не переделывать)
- 5842 надписи с текстом; 7796 токенов; 5029 типов; гапаксы 78%;

- Операторы профилируются: mi «я» ×170 (89% начальная — «говорящие

  «X-al + clan/sec» ×11.

## План (из ROADMAP.md; выполнять поэтапно, с отчётом после каждого)
1. **Заморозка корпуса**: единый etr_corpus.pkl из трёх CSV (дедупликация
   по ID! Larth и ETP пересекаются), нормализация транслитерации (θ/χ/φ/ś,
   пунктуация, лакуны […]), валидация против ручной мини-выборки (~50
   надписей вручную сверить с ETP-онлайн), supplement-механизм. Отчёт §0.
2. **Реестр операторов v1** (известные слова по глоссарию с источниками) +
   слот-шаблоны эпитафий/посвящений; LOO-валидация шаблонов ПО ПЕРЕВОДАМ
   (метрика!).
3. **Калибровка морфем**: суффиксы/энклитики с семейным контролем (p_max)
   и репликацией по городам И векам (датировки есть!).
4. **Дистрибутивная семантика с учителем**: признаковые представления слов
   (слоты, соседи, операторные контексты) → предсказание семантического
   класса; train/test на переведённых; отчёт с точностью по классам;
   затем ранжированные гипотезы для непереведённых слов.
5. **Liber Linteus**: добыть транслитерацию → слот-разбор ритуального
   календаря (движок по образцу formula_template.py из LA).
